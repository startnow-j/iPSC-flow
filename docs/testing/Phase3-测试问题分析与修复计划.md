## 二、发现的问题清单

### P0 — 严重（必须立即修复）

| # | 测试项 | 问题描述 | 根因分析 | 修复方案 |
|---|--------|---------|---------|---------|
| **B1** | F-等 / #3 | **产品线访问控制未生效**：非归属产品线的用户能看到并进入批次详情。例如王QA（仅细胞产品线）能看服务项目批次，李三（无基因编辑权限）能看基因编辑批次 | 批次列表API `/api/batches`、批次详情API `/api/batches/[id]`、状态统计API `/api/batches/status-stats` 均未按用户 `UserProductLine` 过滤数据。`canOperate()`/`canManage()` 仅用于写操作 | 3个读API增加产品线过滤；批次详情页增加 403 守卫 |
| **B2** | B3-1 | **非指定质检员可执行质检**：批次已指定质检员，但其他有QC角色（非该批次指定）的用户也能开始质检并提交 | transition API 和 `getAvailableActions()` 仅检查用户是否有 QC 全局/产品角色，**从未校验 `batch.qcOperatorId === currentUser.id`** | 在 transition API 中增加 `qcOperatorId` 校验；在 `getAvailableActions()` 中过滤非指定质检员的QC操作 |
| **B3** | B3-3 | **QC表单为空可以提交**：质检记录无任何检测数据即可提交合格判定，并自动生成CoA | QC表单提交时未校验 `testResults` 是否为空或有意义数据 | QC提交API增加必填字段校验（至少一项检测结果） |
| **B4** | #4 | **冻存细胞支数计算错误**：输入 `0.5×10⁶` 每支，总细胞 10,000,000，显示 2 支（应为 20 支） | `harvest-form.tsx` 中 `parseVialSpec()` 的正则 `/(\d+)\s*[×xX*]\s*10\^(\d+)/` 使用 `\d+`（仅匹配整数），无法匹配 `0.5`，错误匹配到 `5`，解析为 5,000,000 而非 500,000 | 将正则改为 `/(\d+(?:\.\d+)?)\s*[×xX*]\s*10\^(\d+)/` |
| **B5** | B5-1 | **报废对话框未弹出**：点击报废操作无反应 | 前端报废按钮的 `onClick` 未正确触发对话框，或对话框组件渲染/调用方式有误 | 检查 `batch-status-overview.tsx` 中报废按钮逻辑 |

### P1 — 高优先级（影响核心业务流程）

| # | 测试项 | 问题描述 | 根因分析 | 修复方案 |
|---|--------|---------|---------|---------|
| **B6** | B2-7 | **冻存后仍可添加传代记录**：收获冻存操作完成后，扩增培养页面仍允许新增传代轮次，且页面底部仍显示"提交质检"按钮 | 任务完成状态未正确传递到前端 UI，或已完成任务列表仍渲染了表单入口 | 前端判断：如果后续任务（如 HARVEST）已完成，EXPANSION 不再允许新增轮次 |
| **B7** | B2-7 | **冻存后代次号自动递增错误**：冻存后再次进行扩增操作，质检的细胞代次自动增加，导致已冻存细胞的代次也被错误更新 | 传代表单中代次号自动计算逻辑（`passage_to = passage_from + 1`）读取的是最新记录的 passage_to，未考虑冻存记录的代次 | 冻存时记录当前代次并锁定；后续传代应独立计算 |
| **B8** | B2-7 | **分化细胞代次不应自动递增**：分化细胞代次应独立计算（P0 起算），不应沿用 iPSC 传代代次 | 分化诱导表单使用了与扩增培养相同的代次逻辑 | 分化诱导表单的代次改为手动输入，默认 P0 |
| **B9** | B3-5 | **SUPERVISOR 无质检返工按钮**：分化产品质检提交后，SUPERVISOR 找不到"返工"按钮 | 可能是特定产品类别（如 NPC）的状态机可用动作缺少 `rework`，或前端按钮渲染有条件遗漏 | 检查 `getAvailableActions()` 对 QC_IN_PROGRESS 状态返回的动作列表 |
| **B10** | B3-4 | **质检提交后无法返回修改**：质检结果提交后发现为空，无法回到 QC_IN_PROGRESS 重新填写 | `rework` 操作仅对 SUPERVISOR 可见；且"重新生成 CoA"按钮不等于修改质检结果 | 在 QC_PASS 状态下增加"撤回质检"操作，或允许 SUPERVISOR 返工后重新填写 |
| **B11** | B3-6 | **未找到过程采样入口**：扩增培养表单中无采样记录添加功能 | 采样记录组件 `sampling-record.tsx` 可能未集成到 expansion-form.tsx 中 | 在 expansion-form 中集成采样记录组件 |
| **B12** | B3-2 | **OPERATOR 能看到质检界面**：非质检人员可以看到 QC Tab 内容（虽然不能提交） | 前端 QC Tab 未做角色隐藏 | QC Tab 仅对 QC 角色或 SUPERVISOR 可见 |
